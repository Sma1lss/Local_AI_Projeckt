# history

Дисковое хранилище истории C++ Router.

## Формат

Для каждого `chat_id` создается отдельная папка:

- `models/<model>.jsonl` - все ответы конкретной модели;
- `judge.jsonl` - финальные judge-ответы;
- `meta.json` - метаданные чата.

Каждая JSONL-запись содержит:

- `turn_id`;
- `ts`;
- `question`;
- `answer`;
- `model`;
- `ok`;
- `error`.

## Поведение

- число записей обрезается по `HISTORY_MAX_TURNS`;
- при новом чате создается новый `chat_id`;
- при повторном запросе подтягивается контекст последних ходов из `judge.jsonl`.
