# data/docs

Каталог исходных документов для RAG.

## Что сюда класть

- проектную документацию;
- технические заметки;
- markdown/text-файлы по предметной области;
- при необходимости исходники, которые должны участвовать в retrieval.

## Что происходит дальше

После пополнения каталога нужно заново построить индекс через `services/index_faiss.py`.
